# 数据库执行安全清单

## 执行前

1. 明确目标环境（开发 / 测试 / 预发），避免误连生产库。
2. 先跑 `SELECT` 验证命中范围，再执行 `UPDATE/DELETE`。
3. 写操作必须带业务条件；如果是主键批量更新，先抽样核对 3~5 条。
4. 涉及删除时，先确认是否需要逻辑删除替代物理删除。

## 执行中

1. 通过 `scripts/run_sql.py` 执行，保持一致的安全护栏。
2. 写操作强制携带 `--allow-write --confirm CONFIRM_WRITE`。
3. `UPDATE/DELETE` 无 `WHERE` 默认会被拦截，除非显式允许全表写。

## 执行后

1. 对写操作补一次反向 `SELECT`，确认影响范围符合预期。
2. 记录执行 SQL、执行时间、影响行数，方便问题追踪。
3. 如果是高风险变更，补充回滚 SQL 或数据恢复方案。
